# Now

## Current Stage
目前處於：

- post-interview
- early job onboarding
- building stable execution rhythm

這個階段的重點不是擴張主題，而是先建立：

- 工作上下文
- 可持續的輸出節奏
- 不容易漂移的知識邊界

## Primary Goals
目前 vault 應優先服務這幾件事：

- 加速新工作上手與領域理解
- 維持生活管理、健康管理與心理狀態的基本盤
- 沉澱工作中反覆出現的 decision / checklist / pitfalls
- 讓學習內容更貼近當前職責，而不是沿用面試時期的資料結構
- 保留 English / trading / life，但不讓它們搶走主線

## What This Vault Is For Now
當前這個 vault 的主要角色是：

- 工作與生活的本地知識層
- 交付、學習、復盤的支撐系統
- 不是面試題庫延伸版
- 也不是無限制的資料收藏箱

## Active Focus
- 新工作 onboarding 與 domain ramp-up（system design 導向，不需刷題）
- 日常健康追蹤：減脂、喝水、營養品
- 推動程式以外的活動：吉他、畫畫 / AI 生圖、做遊戲
- 股票研究能力與英文能力（在 Habit Tracker Game 中追蹤）
- 心理狀態管理與低狀態恢復

## Deprioritized For Now
以下內容可以保留，但不應主導結構：

- LeetCode 刷題（新工作不考這塊）
- backend interview drill
- system design 題庫式整理
- 純收藏型閱讀追蹤
- 沒有明確使用場景的知識摘錄

## Capture Rules
新增內容前，先問：

- 這是否服務目前工作階段？
- 這是可重用知識，還是只是來源快照？
- 這應該進 `03_notes/`、`04_playbooks/`，還是其實不值得進庫？
- 三週後我還會需要它嗎？

如果答案偏弱，就先不要擴張結構。

## Success Signals
這個 vault 沒有跑歪時，應該看得到：

- 工作中常用的判斷標準逐步沉澱成 note / playbook
- current focus 會隨人生階段更新
- source 持續存在，但不壓過 curated knowledge
- 每個主題都有明確「為什麼現在重要」

## Review Trigger
以下情況應更新這頁：

- 工作階段改變
- 主線目標改變
- 某個主題突然佔用太多注意力
- vault 開始大量收集但很少回用
